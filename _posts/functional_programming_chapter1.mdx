## 용어 정의

### 부수 효과 (side effect)
부수 효과: 함수가 리턴값 이외에 하는 모든 일을 말한다. (ex. 이메일 보내기, 전역 변수 수정하기)
부수 효과는 함수를 호출할때마다 실행되기 때문에 문제가 될 수 있다. (ex. 함수의 리턴값이 필요해서 호출했지만 의도하지 않게 외부의 값을 수정한다거나)
함수형 프로그래머는 불필요한 부수효과를 가능한 사용하지 않으려 한다.

### 순수 함수 (=수학 함수)
순수 함수: 인자에만 의존하고, 부수 효과가 없는 함수, 같은 인자를 넣으면 항상 같은 결과값이 나온다.

## 함수형 프로그래밍의 정의
1. 수학 함수(=순수함수)를 사용하고 부수 효과(side effect)를 피하는것이 특징인 패러다임
2. 부수 효과 없이 순수 함수만 사용하는 프로그래밍 스타일

정의에 따르면 함수형 프로그래머는 순수 함수만 사용해야 할 것 같지만, 실제 함수형 프로그래머는 부수효과와 순수하지 않은 함수를 사용한다!
위 정의는 실용적인 측면에서 문제가 있다.

### 부수 효과는 필요합니다.
부수효과는 소프트웨어를 실행하는 이유이다.
이메일을 전송하지 않는 이메일 전송 소프트웨어가 무슨 의미가 있을까?
정의에는 부수효과를 완전히 쓰지 말라는 것처럼 되어 있지만, 필요할때는 써야 한다.

### 함수형 프로그래밍은 부수효과를 잘 다룰 수 있습니다.
함수형 프로그래머는 부수효과가 실제로 필요하지만 문제가 될 수 있다는 것을 알기 때문에, 부수 효과를 잘 다룰 수 있는 도구를 사용한다.
(도구가 어떤것인지는 뒤 챕터에서 설명)

## 액션과 계산, 데이터 구분하기 - 함수형 프로그래머가 코드를 바라보는 방법
많은 함수형 프로그래밍 기술과 개념은 코드를 액션, 계산, 데이터로 구분하는 것으로부터 시작한다.

### 액션
액션은 호출시점과 횟수에 의존한다. 따라서 호출할때 조심해야 한다.
ex. 긴급한 메일을 오늘 보내는 것과, 다음주에 보내는 것은 완전히 다르다. 같은 메일을 10번 보내는 것과 한 번 보내는 것 또는 보내지 앟는 것 역시 다르다.

액션의 도구
- 시간이 지남에 따라 안전하게 상태를 바꿀 수 있는 방법
- 순서를 보장하는 방법
- 액션이 정확히 한 번만 실행되게 보장하는 방법

### 계산
입력값으로 출력값을 만드는 것.
언제 어디서 같은 입력값으로 같은 결과값이 나오며, 외부에 영향을 주지 않는다. 따라서, 계산은 테스트하기 쉽고 언제든지 몇 번을 불러도 안전하다.

계산의 도구
- 정확성을 위한 정적 분석
- 소프트웨어에서 쓸 수 있는 수학적 지식
- 테스트 전략

### 데이터
이벤트에 대해 기록한 사실, 알아보기 쉬운 속성으로 되어 있고, 실행하지 않아도 데이터 자체로 의미가 있음.

데이터의 도구
- 효율적으로 접근하기 위해 데이터를 구성하는 방법
- 데이터를 보관하기 위한 기술
- 데이터를 이용해 중요한 것을 발견하는 원칙
